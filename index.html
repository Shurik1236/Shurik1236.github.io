Итак, для начала давайте расскажу что понимается под словами "краш" и "лог". 

КРАШ это текстовый файл лежащий в minecraft\crash-report. Выглядит так:
crash-2016-09-08_09.48.32-client.txt или crash-2016-09-28_18.31.26-server.txt.

ЛОГ это текстовый файл который лежит в .minecraft\logs.
На TL нам нужно содержимое fml-client-latest.log, на MultiMC latest.log.

НЕ НАДО ПРИСЫЛАТЬ СКРИНШОТЫ ЭТИХ ФАЙЛОВ ИЗ БЛОКНОТА!!!
НЕ НАДО ПРИСЫЛАТЬ НАЧАЛО ИЛИ КОНЕЦ, НУЖЕН ПОЛНЫЙ ФАЙЛ!!!
НЕТ СМЫСЛА ПИСАТЬ "Я установил это и не работает!" ЭТОГО НЕ ДОСТАТОЧНО ЧТОБЫ ПОНЯТЬ ПРИЧИНУ!
НЕ НАДО ПРИСЫЛАТЬ ДОЛБАНЫЙ diagnostic.log! ЧИТАЙТЕ КРАШ!

Теперь открываем блокнотом эти файлы и копируем ВСЁ и вставляем на сайт https://pastebin.com.
Нажимаем "Create New Paste". После этого ссылку из адресной строки в пост ВК.
Обычно нужен только КРАШ, если потребуется ЛОГ то это у вас попросят.
Иногда КРАШ не создаётся, поэтому присылаем ЛОГ.

Прежде чем обращатся откройте в Notepad++ файл и посмотрите первую строку краша после "Description:". Об этом в следующем разделе.

Образцы крашей 
java.lang.IllegalArgumentException: Could not get provider type for dimension -1022, does not exist
	Конфликт ID измерений. Удалите проблемный мод и сделайте дамп измерений, после чего смените ID в конфиге проблемного мода. Допустимый предел обычно от -1024 до 1024.
	Красным обозначен проблемный ID.
	Сменить ID просто. Ищи похожее в конфиге:
	  dimensions {
	  # Dimension ID for Barnarda C
	  I:dimensionIDBarnardaC=-1022
	  ...
	  }
	Числа после = и есть ID.
Как делать дамп измерений.

Аналогично и дамп зачарований, зелий...


java.lang.IllegalArgumentException: Duplicate enchantment id! class net.nevermine.assist.AscensionEnchants$1 and class am2.enchantments.EnchantMagicResist Enchantment ID:100
	Стандартный краш при конфликте ID зачарований. Решается так же как и с измерениями. Допустимый предел от 0 до 255.
	Выделенное красным это конфликтующие классы. Что это вам знать не нужно, важно то что это помогает определить какие моды конфликтуют.
	Число в конце помогает узнать проблемный ID.
	Ищи похожее в конфиге:
	  enchantments {
	  I:ench_haste=150
	  I:ench_repair=151
	  }

Для GalaxySpace...

Итак ты решил поставить GalaxySpace и MorePlanets одновременно и получил краш. Вот сейчас я расскажу что делать чтобы всё работало.

Для начала скачайте последнию версию GC: https://ci.micdoodle8.com/job/Galacticraft-1.7/

Потом идём сюда:

config\GalaxySpace\dimensions.conf

config\GalaxySpace\biomes.conf

config\MorePlanets.cfg

B открываем эти файлы в Notepad++


Теперь делаем дамп биомов и измерений. Предварительно убираем оба мода. Как это далать смотри в "Образцы крашей"

Cделали дамп? Отлично смотрим на значения:

biomes {
...
}

Если будут совпадения, меняем число после = на то которое не занято. Тоже самое далем с этим. Это уже измерения. Смотрите дамп измерений.

dimensions {
...
}

Хорошо, теперь выключите повторяющиеся орбитальные станции и планеты в одном из модов.

У GS это всё тут: config\GalaxySpace\dimensions.conf

У MP в config\MorePlanets.cfg

Допустим я буду выключать в MP. В GS и MP дублируюбтся Фобос, Деймос, Венера, Меркурий, Плутон и станции Марса.

У GS это в "general {". Конфиг аналогичный. Только там больше параметров. Но названия практически такие же.

true включено, false выключено.

celestial {
    # If you playing with other GC Add-on you can disable Deimos to prevent crash and conflict.
    B:"Enable Deimos Moon"=true //Демос

    # If you playing with other GC Add-on you can disable Jupiter to prevent crash and conflict. But if disable this option and enabled Jupiter Space Station. Game will getting crash!
    B:"Enable Jupiter Planet"=true //Юпитер, выключаем с Galaxy Additions

    # If you playing with other GC Add-on you can disable Jupiter Space Station to prevent crash and conflict. But if disabled Jupiter and enabled Jupiter Space Station option. Game will getting crash!
    B:"Enable Jupiter Space Station"=true //Станция Юпитера, выключаем с Galaxy Additions

    # If you playing with other GC Add-on you can disable Mars Space Station to prevent crash and conflict.
    B:"Enable Mars Space Station"=true //Марсианская станция

    # If you playing with other GC Add-on you can disable Mercury to prevent crash and conflict.
    B:"Enable Mercury Planet"=true //Меркурий

    # If you playing with other GC Add-on you can disable Phobos to prevent crash and conflict.
    B:"Enable Phobos Moon"=true //Фобос

    # If you playing with other GC Add-on you can disable Pluto to prevent crash and conflict.
    B:"Enable Pluto Planet"=true //Плутон

    # If you playing with other GC Add-on you can disable Venus to prevent crash and conflict.
    B:"Enable Venus Planet"=true //Венера
}

Ещё в конфиге MP в general найдите B:"Enable New Main Menu Screen"=true или в GS B:enableNewMenu=true. Это тоже выключите.

Ах да, выключите новую галактическую карту в GS. Она в core.conf в dimensions. И MP с ней не работает. B:enableNewGalaxyMap=true
